## 📝 Project Summary

**FAQ chatbox - M7** is a fully customized, ultra-modern full-stack web application designed to re-imagine standard customer support into an engaging, interactive, and intelligent conversational system. 

This platform seamlessly integrates a highly optimized frontend interface with administrative tools. **No API key needed!** The application works out-of-the-box with intelligent keyword-based search, browser-native text-to-speech, and local NLP processing. Optionally integrate Google Gemini API for enhanced semantic search and AI features.

## 🚀 Key Use Cases of the Project

The application addresses modern corporate, support, and learning directory needs through several targeted use cases:

### 1. Instant Conversational Support Integration (For Users)
- **Problem**: Traditional FAQ pages are static, boring, and force users to scan endless documents.
- **Solution**: Users chat directly with the **M7 Intelligent Assistant**. The AI parses user searches, extracts matched keywords, and returns answers instantly.
- **Advanced Assistive Helpers**: Includes accessibility features such as **Text-to-Speech (TTS)** voice readbacks and dynamic follow-up recommendations if search confidence is low.

### 2. High-Performance Knowledge Directory (For Users)
- **Problem**: Users want structured, categorized lists to browse when they aren't searching.
- **Solution**: A structured bento grid presents categories clearly. Users can expand accordion listings, rate content helpfulness, and search topics instantly.

### 3. Continuous Learning & Support Escalation (For Admins)
- **Problem**: Unresolved queries often get lost, leading to repetitive questions.
- **Solution**: Any queries that can't be resolved are automatically captured in the admin pool. Admins can view these unhandled questions, convert them into new FAQs, or escalate them to direct support tickets with high, medium, or low priorities.

### 4. Direct Version Auditing & Knowledge Base Rollbacks (For Admins)
- **Problem**: Editing FAQs can lead to errors with no easy way to revert changes.
- **Solution**: Every modification creates a historic delta. Admins can view the complete edit timeline and roll back the entire knowledge base structure to any previous state with a single click.

## 🚀 Installation & Build Pipeline

Initialize and run the platform in your sandbox environment:

```bash
# 1. Populate node_modules
npm install

# 2. Run express development environment server
npm run dev

# 3. Check for typescript type issues and syntax errors
npm run lint

# 4. Build static distribution bundle and server
npm run build

# 5. Launch the compiled production server
npm run start
```

## 🔑 API Key Configuration (Optional)

**The application now runs fully without an API key!** ✅

### Running Without an API Key (Default)
By default, the application will run with all core features enabled:
- ✅ **Chat with Keyword-Based Search**: Searches FAQs using intelligent substring matching and TF-IDF scoring
- ✅ **Text-to-Speech**: Falls back to browser's Web Speech API
- ✅ **FAQ Management**: Create, edit, and delete FAQs
- ✅ **Admin Dashboard**: View analytics, manage tickets, and track user interactions
- ✅ **Support Tickets**: Create and manage customer escalations

### Enabling Advanced AI Features (Optional)
To unlock enhanced capabilities powered by Google Gemini API, set the `GEMINI_API_KEY`:

```bash
# Create or edit .env file
GEMINI_API_KEY=your_gemini_api_key_here
```

**With API Key enabled**, you get:**
- 🚀 **Semantic Vector Search**: Uses embeddings for more accurate FAQ matching (>70% confidence)
- 🎯 **Intelligent NLP Processing**: Advanced sentiment analysis, intent detection, and entity extraction
- 🤖 **Automatic FAQ Generation**: Extract FAQs automatically from uploaded documents
- 🔊 **Premium Text-to-Speech**: Uses Gemini's professional TTS instead of browser speech
- 💡 **Dynamic Answer Generation**: AI generates contextual responses for unmatched queries

### Architecture with Fallbacks
The application intelligently handles both scenarios:

| Feature | Without API Key | With API Key |
| :--- | :--- | :--- |
| **FAQ Search** | Keyword-based (TF-IDF) | Semantic embeddings + keyword fallback |
| **NLP Analysis** | Local rules-based detection | Advanced Gemini NLP |
| **Text-to-Speech** | Browser Web Speech API | Gemini TTS (24000 Hz PCM) |
| **FAQ Extraction** | Disabled | Gemini document parsing |
| **Admin Response** | Basic fallback text | AI-powered summaries |

## 🌐 Environment Variables

Create a `.env` file in the project root (optional):

```env
# Optional: Google Gemini API Key for enhanced AI features
# Leave empty to run with keyword-based search
GEMINI_API_KEY=your_api_key_here

# Server configuration
NODE_ENV=development
PORT=3000
```

If the `GEMINI_API_KEY` is missing or invalid, the application logs:
```
⚠️  Skipping embedding pre-computation - GEMINI_API_KEY is missing or invalid.
✅ Application will work fine without embeddings using keyword-based search.
```
